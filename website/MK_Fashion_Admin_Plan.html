🔴 3. Phase 1 — Shell, Auth & Dashboard (Day 1–2)

Step 1 · Auth Guard & Login Page

  • Create src/lib/adminAuth.ts — custom hook reads sessionStorage.admin_auth
  • Login page at /admin/login — password input with show/hide toggle + animated submit
  • Correct password sets auth cookie → redirect to /admin
  • Admin layout.tsx runs guard — redirects to /admin/login if unauthenticated
  • Password stored in .env.local as NEXT_PUBLIC_ADMIN_PASSWORD

Step 2 · Admin Layout Shell (enhance existing)

  • Rebuild admin/layout.tsx with full sidebar + topbar layout
  • Left sidebar: store logo, nav links with Lucide icons, user avatar + logout at bottom
  • Active route highlighted with purple accent + left border glow
  • Collapsible sidebar (icon-only mode) — state persisted in Zustand
  • Top bar: breadcrumb trail, search icon, notification bell, admin name
  • Framer Motion AnimatePresence wraps pages for smooth transitions

Step 3 · Dashboard Overview (rebuild existing /admin page)

KPI CardMetricAccent Color
💰 Total RevenueAll-time + today's revenueEmerald
🛒 Total OrdersOrders today / pending countIndigo
👗 ProductsTotal active vs draft vs archivedPurple
👥 CustomersTotal + new this monthAmber

Charts & Feed below cards:

  • Area chart — revenue over last 30 days
  • Bar chart — orders per day (last 14 days)
  • Pie chart — order status breakdown (Pending / Processing / Shipped / Delivered)
  • Recent orders feed — last 10 orders with customer name, amount, status badge
  • Top-selling products table — rank, name, image thumbnail, units sold, revenue
  • Low-stock alerts widget — products below threshold

🟡 4. Phase 2 — Product, Order & Customer Management (Day 3–5)

Step 4 · Product Management /admin/products

  • Product list table: Image thumbnail · Name · Category · Price · Stock · Status badge · Actions
  • Search by name · Filter by category, status (Active/Draft/Archived), stock level
  • Bulk actions: Bulk delete · Bulk publish · Bulk archive
  • Row actions: Edit · Duplicate · Delete (with confirm modal)
  • Pagination — 15 products per page
  • "+ Add Product" button → opens /admin/products/new

Step 5 · Product Add/Edit Form /admin/products/[id]

  • Basic Info: Name, Description (rich text), SKU, Brand
  • Pricing: Price, Compare-at price, Cost per item, Taxable toggle
  • Inventory: Stock count, Low stock threshold, Track inventory toggle
  • Variants: Add size/color/material variants with individual stock per variant
  • Images: Multi-image upload with drag-and-drop reorder, set primary image
  • Category: Select from category tree (multi-select for subcategories)
  • SEO: Meta title, meta description, URL slug (auto-generated from name)
  • Status: Draft / Active / Archived toggle
  • Save as Draft · Publish buttons

Step 6 · Category Management /admin/categories

  • Visual nested tree: Parent → Child categories (e.g., Sarees → Silk Sarees)
  • Drag-and-drop reordering within same level
  • Inline add/edit/delete category nodes
  • Category banner image upload
  • Visibility toggle per category (Show/Hide on storefront)

Step 7 · Order Management /admin/orders

  • Table columns: Order # · Customer · Date · Items · Total · Payment status · Delivery status · Actions
  • Filter tabs: All / Pending / Processing / Shipped / Delivered / Cancelled / Refunded
  • Search by order ID, customer name, or email
  • Date range filter · Sort by date, amount
  • Bulk actions: Mark as shipped · Export selected
  • Actions: View detail · Print invoice PDF

Step 8 · Order Detail /admin/orders/[id]

  • Order summary: items, quantities, images, unit prices, subtotal, tax, shipping, total
  • Customer details: name, email, phone, shipping address
  • Order status stepper: Placed → Confirmed → Shipped → Delivered
  • Status update dropdown with timestamp log
  • Add tracking number + carrier name
  • Refund / Cancel order button (with reason modal)
  • Download invoice as PDF
  • Internal admin notes field (private, not visible to customer)

Step 9 · Customer Management /admin/customers

  • Table columns: Avatar · Name · Email · Phone · Total Orders · Total Spent · Joined · Status · Actions
  • Search by name/email · Filter by status (Active/Blocked)
  • Click row → slide-out profile drawer with order history
  • Actions: View · Block/Unblock · Delete
  • Export to CSV (all or filtered customers)

🟢 5. Phase 3 — Analytics, Promotions, Content & Settings (Day 6–7)

Step 10 · Analytics /admin/analytics

Category performanceRevenue by category (Sarees, Lehengas, Kurtis…)
Customer growthNew customers per month cumulative area chart
Top productsBest-selling products table (units + revenue)
Geo breakdownOrders by city/state (table)
  • Date range picker (Last 7d / 30d / 90d / Custom)
  • Export report as CSV or PDF download

Step 11 · Promotions & Discounts /admin/promotions

  • Coupon creator: Code · Type (% or fixed ₹) · Value · Min order · Expiry date · Usage limit · Active toggle
  • Coupon table: code, type, value, uses remaining, expiry, status badge, copy & delete
  • Flash sale widget: Set product(s) on sale with countdown timer + discounted price
  • Free shipping threshold: Set minimum cart value for free shipping

Step 12 · Content Management /admin/content

  • Homepage banner: Upload hero image, set headline, subheadline, CTA link
  • Announcement bar: Site-wide top banner text (sale alerts, policy notices)
  • Featured collections: Choose which categories appear on homepage
  • Sale banners: Upload promotional banners for category pages
  • Live preview toggle · Publish changes button

Step 13 · Reviews Moderation /admin/reviews

  • Review list: product image, product name, customer, star rating, review text, date, status
  • Filter: All / Approved / Pending / Rejected
  • Actions: ✅ Approve · ❌ Reject · 🗑️ Delete
  • Admin reply input field per review
  • Rating analytics: avg rating per product, distribution chart

Step 14 · Settings /admin/settings

  • Store Info: Store name, logo, favicon, contact email, phone, address
  • Currency & Tax: Currency (₹ INR), GST % per category
  • Shipping: Zones by state/pincode, flat rate / free threshold / weight-based
  • Payment Methods: Enable/disable UPI, Cards, COD, Wallets
  • Notifications: New order email, low stock alert, daily summary email
  • Admin Users: Add/remove admins, set roles (Super Admin / Manager / Support)
  • API Keys: Payment gateway keys (masked + reveal toggle)
  • Danger Zone: Clear all sessions / reset store data

🔌 7. API Integration (Frontend ↔ NestJS)

The admin frontend communicates with the NestJS API at apps/api via TanStack Query (already installed). Admin API calls are centralized in src/lib/adminApi.ts.

PageNestJS EndpointMethodPurpose
Dashboard/admin/statsGETKPI cards + chart data
Products list/productsGETPaginated product list
Product form/products/:idGET/POST/PATCH/DELETECRUD single product
Orders list/ordersGETPaginated + filtered orders
Order detail/orders/:idGET/PATCHView + update order status
Customers/customersGET/PATCH/DELETEList + manage customers
Categories/categoriesGET/POST/PATCH/DELETECategory tree CRUD
Analytics/admin/analyticsGETSales & traffic metrics
Promotions/promotionsGET/POST/PATCH/DELETECoupon management
Reviews/reviewsGET/PATCH/DELETEReview moderation
Settings/admin/settingsGET/PATCHStore configuration

During development, all calls are served by src/lib/mockData.ts — no NestJS server needed until Phase 2+.
